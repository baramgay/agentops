# 민간데이터 전처리 핵심 학습 (사무실-데스크탑 PC)

**경남 청년 정착 잠재지역 발굴 분석 프로젝트**에서 축적된 전처리 경험.

## KT 통신 데이터

**신구버전 자동 판별**
- `fname[:6].isdigit()` 로 신(2023~)/구(~2022) 분기
- 구버전 인코딩: utf-8(-sig) — cp949가 아님 (실수하기 쉬운 지점)
- 신버전: `header=None` 필수

**유입/유출 컬럼 순서 반대**
- 유입: `base_date | sgg_cd(목적지) | rsdn_sgg_cd(출발지)`
- 유출: `base_date | rsdn_sgg_cd(경남출발) | sgg_cd(목적지)`
- 방향 혼동하면 순이동 부호가 반전됨 — pandas merge 시 컬럼명 명시 필수

**생활인구 특성**
- 체류인일 스냅샷 누적값 — 동일인 중복 포함, 절대 인구수와 다름
- 보고서에 반드시 이 특성 명시

## KB국민카드 데이터

**age_cd 형식**
- `'2.2024'`(20~24), `'3.2529'`(25~29), `'4.3034'`(30~34), `'5.3539'`(35~39) — 마침표 포함 문자열
- 숫자 비교 아닌 문자열 완전 일치로 필터링

**ms 보정계수**
- `_ms` 접미사 파일이 있으면 무조건 그 파일 우선 (이미 보정됨)
- 2018~2022 통합본: `cnt/=factor`, `amt/=factor` 보정 필수
- 보정 후 2018~2025 연속 추이 확보 — 연도별 보정계수 0.141~0.153으로 안정적

**헤더 규칙**
- 거의 모든 파일 `header=None` — 2018~2022 통합본만 예외

## KCB 신용융합 데이터

**누적 구조 — 최신 폴더만 사용**
- `(KCB)Your Region청_데이터_YYYYMMDD` 중 가장 최신 폴더만 사용
- 인코딩: cp949 / 구분자: pipe(`|`)

**1인당 vs 총합 혼동 주의 (핵심 함정)**
- **이미 1인당 값**: MA00001(추산소득), MS00002(신용점수) → `/MP00001` 나눗셈 금지
- **총합**: ML00009(대출잔액), MC00006(카드이용금액) → `/MP00001`로 환산 필요
- 이 구분을 잘못 알면 이중 나눗셈 버그 발생 → 명세서 xlsx 직접 확인으로 검증
