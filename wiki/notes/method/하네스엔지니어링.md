# 결론 - Harness Engineering

## 핵심 정의

Harness Engineering은 AI에게 더 좋은 프롬프트를 작성하는 기술이 아니다.

AI가 실수할 수 없는 작업 환경을 설계하는 기술이다.

즉,

- Prompt Engineering = AI에게 잘 말하는 기술
- Context Engineering = AI에게 충분한 정보를 제공하는 기술
- Harness Engineering = AI가 안정적으로 일하도록 시스템을 구축하는 기술

---

## 왜 중요한가

AI는 이미 충분히 똑똑하다.

문제는 지능이 아니라 일관성이다.

따라서 앞으로의 경쟁력은 더 좋은 모델을 사용하는 것이 아니라,

- 규칙을 어떻게 관리하는가
- 문서를 어떻게 구조화하는가
- 검증을 어떻게 자동화하는가
- 역할을 어떻게 분리하는가
- AI가 어떻게 스스로 확인하게 만드는가

에 의해 결정된다.

---

## 핵심 원칙

### 1. 설명보다 구조

AI에게 긴 프롬프트를 주기보다 올바른 작업 구조를 제공한다.

### 2. 규칙보다 강제

"하지 마"라고 말하지 않는다.

시스템적으로 못 하게 만든다.

### 3. 문서는 지도

AGENTS.md 또는 CLAUDE.md는 백과사전이 아니라 네비게이션 역할을 해야 한다.

### 4. 검증 가능한 환경

AI가 직접

- 테스트
- 로그
- UI
- 메트릭

을 확인할 수 있어야 한다.

### 5. 사람 개입 최소화

최종 목표는 코드 생성이 아니라 인간의 검수 비용 감소다.
