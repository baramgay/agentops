# 위키 사용 규칙 (사람 + LLM 공용)

## 노트 원칙
- **원자성**: 노트 1개 = 개념 1개. 제목이 곧 ID(파일명 = 슬러그).
- **증류**: 원문 덤프 금지. "결론 · 이유 · 적용법" 중심으로 압축해서 쓴다.
- **링크 적극**: 관련 노트는 `[[슬러그]]`로 연결. 아직 없는 노트 링크도 OK(만들 거리 표시).
- **표기 규칙**: 한자·일본어 금지(순수 한글). [[feedback-no-hanja]] 준수.

## frontmatter 컨벤션
```yaml
---
name: 슬러그(파일명과 동일)
type: project | feedback | reference | method | decision | moc
domain: 경남부동산 | 이음지도 | agents시스템 | ...
tags: []
updated: YYYY-MM-DD
---
```

## 폴더 구조
- `00_홈.md` — 진입 허브
- `MoC/` — 도메인별 지도(인덱스) 노트
- `notes/` — 원자적 지식 노트
- `_tools/` — 부트스트랩·유지보수 스크립트 (옵시디언 그래프에서 제외 권장)
- `_attachments/` — 첨부(이미지 등)

## 검색 = "탐색" (RAG 아님)
1. **홈/MoC**에서 시작 → 전체 지도 파악
2. `[[링크]]` 따라 필요한 노트만 통째로 읽기
3. 그래도 못 찾으면 **전문검색**(ripgrep / 옵시디언 Ctrl+Shift+F)

## 큐레이션 루프 (Claude·에이전트)
- 작업 **시작**: 해당 MoC를 먼저 읽고 링크 따라 맥락 확보
- 작업 **끝**: 새로 배운 것을 원자적 노트로 저장 → 기존과 **중복 병합** → MoC에 링크 추가
- Vault 전체를 한 번에 로드하지 않는다(컨텍스트 절약). 인덱스 + 타깃 노트만.

## 주의
- `_tools/bootstrap_from_memory.py`는 **1회성 씨앗**. 재실행 시 `notes/`가 메모리 원본으로 덮어써짐 → 옵시디언 편집분 손실. Phase C(양방향 동기화) 설계 전까지 재실행 금지.
