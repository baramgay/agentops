# 경남분석보고서

> **한자·일본어 사용 절대 금지**
> - "분석"은 반드시 순수 한글(U+BD84 + U+C11D)로만 표기한다. 한자 혼용(석 = U+6790) 금지.
> - 일본어 문자·문장 사용 금지.
> - 적용 범위: 본문 텍스트, 파일명, 캡션, 코드 내 문자열 — 예외 없음.

Your Organization 분석 결과를 정식 분석보고서(20~40p) 양식(표지-요약-개요-설계-프로세스-결과-결론)으로 편집하는 스킬. 분석 주제가 바뀌어도 동일한 문서 골격을 재사용한다. 간략 결과보고(2~3p)는 '정보통신담당관실-결과보고' 스킬을 사용한다.

트리거: "보고서 작성", "결과보고 양식", "분석보고서 만들어줘", "경남분석보고서", "/경남분석보고서" 등 Your Organization 분석 결과물 보고서 편집 요청 시 사용.

---

## 입력 요구사항

사용자는 아래 중 하나 이상을 제공해야 한다.

| 항목 | 필수 | 설명 |
|------|------|------|
| 분석 주제명 | 필수 | 보고서 제목에 들어갈 주제 (예: "고령층 의료접근성 취약지 발굴 분석") |
| 분석 결과 데이터 | 필수 | PDF, Word, 엑셀, 또는 텍스트 형태의 분석 결과물 |
| 과제 구분 | 선택 | "자체발굴" / "수요대응" / "정기분석" 중 택 1 (기본값: 자체발굴) |
| 작성자 | 선택 | 기본값: "Your Organization 팀장 이 현 기" |
| 시군별 개별 페이지 여부 | 선택 | 18개 시군 각 1p 개별 분석 페이지 포함 여부 (기본: 미포함) |

## 표준 보고서 구조

### 전체 흐름 (7개 블록)

```
[1] 표지                    — 1p
[2] 요약                    — 1p
[3] Ⅰ 분석개요              — 1p
[4] Ⅱ 분석설계              — 1~2p
[5] Ⅲ 분석 프로세스          — 2p
[6] Ⅳ 분석결과              — 본문 (가변)
[7] Ⅴ 결론 및 시사점         — 2~3p
```

총 페이지 수 가이드: 최소 20p, 일반 25~30p, 시군별 개별 페이지 포함 시 40p 내외.

### [1] 표지 (1p)

> 상세 규칙: [01_표지.md](01_표지.md)

- 페이지 중앙 배치
- 상단: 과제 일련번호 "01" (2자리, 굵은 글씨)
- 중앙: 분석 주제명 2~3줄 (큰 글씨, 줄바꿈 허용)
- 하단: 빈 여백
- 배경: 단색 또는 기관 CI 적용 가능

### [2] 요약 (1p)

> 상세 규칙: [02_요약.md](02_요약.md)

구성 순서:

```
01
[분석 주제명 전체]
[과제 구분]                          ← "자체발굴" / "수요대응" 등
[작성자 소속 및 이름]
요  약
[요약 본문 — 200~400자]
```

요약 본문 작성 규칙:
- 분석 대상·범위 → 주요 결과 → 유형/등급 구분 결과 → 정책 방향 순서로 기술
- 구체적 수치(건수, 비율, 지역명) 포함
- 3~4개 문단, 총 200~400자

### [6] Ⅳ 분석결과 (본문 — 가변)

> 상세 규칙: [06_분석결과.md](06_분석결과.md)

#### 제목 체계

| 수준 | 표기 | 예시 |
|------|------|------|
| 대섹션 | `Ⅳ` | Ⅳ 분석결과 |
| 중섹션 | 아라비아 숫자 `1`, `2`, `3`... | 1 고령 의료 수요·공급 현황 |
| 소섹션 | 한글 가나다 `가`, `나`, `다`... | 가 시군별 고령 생활인구 현황 |

#### 페이지 단위 구성 (기본 패턴)

```
[소섹션 제목]
• 해석 불릿 5~6개
[그림] 캡션 제목
```

- **1페이지 = 불릿 5~6개 + [그림] 1개** 가 기본 밀도
- 불릿이 많거나 표가 추가되면 2p로 분할
- 한 페이지에 [그림] 최대 2개 (2개일 경우 불릿 생략 또는 최소화)

#### 불릿 해석 규칙

- 매 불릿은 관측 사실 또는 해석을 1~2줄로 기술
- "~으로 나타남", "~을 보여줌", "~을 시사함", "~이 확인됨" 등 객관적 서술체
- 마지막 1~2개 불릿은 정책적 함의 또는 활용 가능성 언급
- 수치(건수, 비율, 순위)를 반드시 1개 이상 포함

#### [그림]/[표] 캡션 규칙

```
[그림] 시군별 고령 생활인구 현황
[그림] 취약점수 단계구분도
[표] 활용 데이터 목록
[표] 시군별 취약비율 상위 현황
```

- 캡션 접두어: `[그림]` 또는 `[표]` (대괄호 포함)
- 캡션 위치: 그림/표 **아래**
- 번호 없이 제목만 표기 (그림 1, 그림 2 형식 사용하지 않음)
- 캡션 문체: 명사형 종결 ("~현황", "~분포", "~비교", "~추이")

#### 표 서식 규칙

- 헤더 행 배경색 구분
- 컬럼: 왼쪽 정렬(텍스트), 오른쪽 정렬(숫자)
- 단위는 헤더에 괄호로 표기: (km), (명), (%), (만원)
- 하단 캡션: `[표] 제목`

#### 시군별 개별 페이지 (선택적)

시군별 개별 분석이 필요한 경우 아래 구조를 적용:

```
[총괄 페이지 — 경남 전체 지도/차트 1~2p]

○ 거제시          ← 1p
○ 거창군          ← 1p
○ 고성군          ← 1p
...
○ 합천군          ← 1p
```

- 시군 순서: 가나다순 (거제-거창-고성-김해-남해-밀양-사천-산청-양산-의령-진주-창녕-창원-통영-하동-함안-함양-합천)
- 각 페이지: `○ 시군명` 제목 + 해당 시군 핵심 차트 2~4개 + 불릿 요약 3~5개
- 차트 배치: 2×2 격자 또는 상하 2단

### [7] Ⅴ 결론 및 시사점 (2~3p)

> 상세 규칙: [07_결론및시사점.md](07_결론및시사점.md)

3개 하위 블록:

```
Ⅴ결론 및 시사점

▢분석 결과 종합
y  (핵심 결과 요약 — 5~8개 불릿)

▢정책적 시사점
y  (정책 제언 — 5~8개 불릿)

▢향후 과제
y  (데이터 갱신, 지표 고도화, 후속 분석 등 — 5~10개 불릿)
```

- 불릿 접두어: `y` (소문자 영문 y — 기존 한글 양식 준용)
- 각 불릿은 2~4줄 분량의 서술형 문장
- "분석 결과 종합"은 Ⅳ의 핵심 수치를 재인용
- "정책적 시사점"은 유형별/지역별 맞춤 정책 방향 제시
- "향후 과제"는 데이터 한계, 추가 분석 방향, 모니터링 체계 등

## 편집 규칙 요약

### 문체

| 구분 | 문체 |
|------|------|
| 요약 | "~되었다", "~확인되었다" (과거형 서술체) |
| 개요·설계·프로세스 | "~임", "~함", "~있음" (명사형 종결) |
| 분석결과 불릿 | "~으로 나타남", "~을 보여줌" (관찰 서술체) |
| 결론 | "~필요", "~과제" (제언형) |

### 페이지 밀도 가이드

| 섹션 | 페이지당 요소 |
|------|--------------|
| 표지 | 제목만 |
| 요약 | 200~400자 본문 |
| 개요 | 소제목 3개 + 불릿 12~15개 |
| 설계 | 소제목 3개 + 불릿 10~12개 + 표 1개 |
| 프로세스 | 대항목 2개 + 소항목 4~6개 + 불릿 15~20개 (2p 합산) |
| 분석결과 (1p당) | 불릿 5~6개 + [그림] 1개 |
| 결론 (1p당) | ▢ 1~2개 + y불릿 5~8개 |

### 색상·서식 규칙

- 대섹션 제목(Ⅰ~Ⅴ): 좌측 컬러 바 또는 배경 강조
- 중섹션 번호(1, 2, 3): 원형 또는 사각 배지 스타일
- 소섹션(가, 나, 다): 들여쓰기 + 볼드
- 불릿: `•` (분석결과/개요) 또는 `y` (결론)
- 표 헤더: 배경색 구분
- 강조 수치: 볼드 처리

### 페이지 번호·머리글·바닥글

- 표지: 페이지 번호 없음
- 요약 이후: 바닥글 중앙에 아라비아 숫자 페이지 번호
- 머리글: 없음
- 바닥글: 페이지 번호만 중앙 배치

## 출력 형식

기본 출력: **Word(.docx)** 파일
- python-docx 라이브러리 사용
- A4 세로, 여백 상하좌우 2cm
- 본문 폰트: 맑은 고딕 10pt, 제목 14~18pt
- 차트는 matplotlib으로 생성 후 이미지 삽입
- 차트 삽입 시 cx/cy 비율 반드시 함께 조정 (세로 찌그러짐 방지)
- 파일명: `{분석주제}_분석보고서.docx`
- 저장 경로: 해당 분석과제 폴더 내


## 주의사항

- 이 스킬은 **문서 구조와 편집 규칙만 정의**한다. 분석 로직, 수치 해석, 정책 제언 문장의 자동 생성은 범위 밖이다.
- 사용자가 제공한 분석 결과물의 내용을 표준 구조에 **재배치**하는 것이 핵심 역할이다.
- 차트 스타일(색상, 폰트, 레이블)은 분석 스크립트에서 이미 생성된 것을 그대로 사용한다.
- 섹션별 상세 규칙이 필요하면 위 하위 파일을 참조한다. 메인 파일과 하위 파일이 충돌하면 하위 파일이 우선한다.
- "분석"은 반드시 순수 한글(U+BD84 + U+C11D)로만 표기한다. 한자 혼용(석 = U+6790) 절대 금지. 파일명, 캡션, 코드 내 문자열 모두 해당. (최상단 규칙 참조)

## HTML 구현 가이드 (CSS + 구조)

### 페이지 분리
각 블록: `<div class="page">`, A4 기준: width 794px, min-height 1123px, padding 60px 70px

### 색상 팔레트
| 용도 | 색상 |
|------|------|
| 대섹션 콼러바 | #1a4080 |
| 표지 배경 | #f5f5f5 또는 #ffffff |
| 표 헤더 배경 | #e8edf5 |

### 대섹션 제목 CSS
```css
.section-header { border-left: 5px solid #1a4080; padding-left: 12px; font-size: 18px; font-weight: bold; }
```

### 불릿
- Ⅰ~Ⅳ: `•` (U+2022), 10~11pt
- Ⅴ 결론: `y` 소문자 y 불릿, 2~4줄
- `padding-left: 1.4em; text-indent: -1.4em`

### ▢ 블록 헤더 (결론 섹션)
```css
.block-header::before { content: "a2  "; }
.block-header { font-weight: bold; font-size: 13px; margin: 20px 0 8px; }
```

### 캡션 (핵심)
번호 없음: `[그림] 컨션 제목` / `[표] 컨션 제목`
```css
figcaption::before { content: "[그림] "; font-weight: bold; }
figcaption { text-align: center; font-size: 9pt; color: #444; margin-top: 4px; }
```

### 표지 HTML
```html
<div class="page cover-page" style="background:#f5f5f5">
  <div class="cover-center">
    <div style="font-size:48px;font-weight:900;color:#333">01</div>
    <div style="font-size:32px;font-weight:bold">경남의 세컨드 라이프형</div>
    <div style="font-size:32px;font-weight:bold">귀촌지 찾기 분석</div>
  </div>
</div>
```
