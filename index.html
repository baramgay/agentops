06 · 이슈 트래커 native — v1.0
외부 도구 없이 agent 시스템 내장 이슈 트래커. 데이터는 issues.json에 서버 영구 저장되고, API 서버·브라우저 UI·CLI 세 경로로 동시 접근 가능합니다.
저장 구조
issues.json
현재 상태 스냅샷. GNI-N 시퀀스 카운터 포함. 원자적 쓰기(tmp→rename)로 충돌 방지.
issues_history.jsonl
append-only 감사 로그. 생성·수정·코멘트·삭제 이벤트 전부 기록. 수정 불가.
localStorage (폴백)
API 서버 미실행 시 브라우저 캐시로 기본 UI 동작 유지. 서버 복구 시 자동 재동기화.
이슈 스키마
id — GNI-N (서버 자동 발번, 순차 증가)
title — 제목 (필수)
description — 상세 내용
status — backlog / todo / in_progress / in_review / done / cancelled
priority — urgent / high / medium / low
assigneeId — 33개 에이전트 ID 중 선택
labels — 쉼표 구분 자유 태그
comments — [{author, text, ts}] 배열
created / updated — ISO 타임스탬프
REST API 엔드포인트 (7종)
GET/api/issues목록 조회. 파라미터: status · assignee · priority · q(검색)
GET/api/issues/{id}단일 이슈 조회. 없으면 404
POST/api/issues이슈 생성. 서버가 GNI-N 발번. body: title·description·priority·assigneeId·labels
PATCH/api/issues/{id}부분 수정. status·priority·assigneeId·title·description·labels 중 원하는 필드만
DELETE/api/issues/{id}soft delete — status를 cancelled로 변경. 데이터 보존.
POST/api/issues/{id}/comments코멘트 추가. body: author · text
GET/api/issues/statsKPI 통계. total · active · by_status · by_priority · by_assignee
CLI 사용법 (scripts/issue_create.py)
# 이슈 생성
python scripts/issue_create.py "제목" "내용" [agent_id] [priority]
# 목록 조회 (status 필터 선택)
python scripts/issue_create.py --list [in_progress|backlog|done|...]
# 상세 조회
python scripts/issue_create.py --get GNI-N
agent 작업과 자동 연동 (GNI-N 전이)
update_status.py task 인자에 GNI-N을 포함하면 이슈 상태가 자동으로 전이됩니다.
working → 이슈 in_progress
review → 이슈 in_review
done → 이슈 done
예: python scripts/update_status.py data-collector working "GNI-5 공공데이터 수집" → GNI-5 자동 in_progress 전이
※ 이미 done·cancelled 상태인 이슈는 자동 reopen 하지 않습니다.
WebSocket 실시간 동기화
이슈 생성·수정·삭제·코멘트 발생 시 서버가 모든 WS 클라이언트에 {type: "issue_update"} 브로드캐스트 → 여러 탭·세션에서 새로고침 없이 칸반 자동 갱신.
칸반 컬럼 (5단계)
⬜백로그(backlog)
📋할 일(todo)
⚡진행 중(in_progress)
🔍검토 중(in_review)
✅완료(done)
💡 단순 작업은 이슈 생성 불필요 — update_status.py만 사용. 이슈는 추적이 필요한 복합 작업·버그·개선 요청에 활용하세요.

🔗 GitHub Issues 양방향 동기화 (GNI ↔ Issues)
이슈 트래커의 GNI-N 이슈와 GitHub Issues를 서버(api_server)가 양방향 동기화합니다. 브라우저 직접 호출은 CORS/인증으로 불가하므로 로컬 API 서버 경유입니다.
① PAT 설정 (최초 1회)
github.com → Settings → Developer settings → Fine-grained PAT (repo: your-github-username/agent, Issues: Read/write) → config.local.json의 github.token에 저장(gitignore) 또는 환경변수 GITHUB_TOKEN.
② 동기화 실행
sync 탭 또는 이슈 탭의 "☁️ GitHub 동기화" 버튼(수동), 또는 github.sync_interval_min(기본 15분, 0=수동만) 주기 자동 동기화. CLI: python scripts/github_sync.py --status / --sync.
③ 매핑·충돌
상태→open/closed + gni:status/priority/assignee 라벨, 본문 <!-- gni-id:GNI-N --> 마커로 링크. 충돌은 최신 수정 우선(last-write-wins), GitHub 네이티브 open/closed가 stale 라벨보다 우선. 댓글도 양방향 동기화.
④ 페이퍼클립 연동
에이전트 지시 시 "이슈로 등록" 체크 → GNI 이슈 자동 생성·연결, 상태 전환(working→done)에 따라 자동 전이 → 다음 동기화에 GitHub 반영.

에이전트 위치 도트: 작업 중(초록) / 검토(보라) / 기타(파랑).
흰색 테두리 사각형 = 현재 카메라 뷰포트. 1초마다 자동 갱신.

14 · 절대 규칙 요약 (CLAUDE.md)
✅ 매 작업 필수 절차 (생략 불가)
STEP 0 · CLAUDE.md 빠른 참조표로 담당 에이전트 결정
STEP 1 · update_status.py working "[내용]"
STEP 2 · 작업 수행
STEP 3 · update_status.py done "[완료]"
❌ 절대 금지
한자·일본어 사용 (텍스트·코드·파일명 모두)
에이전트 없이 임의 진행
지휘 체계 건너뛰기 (OC → 리드 → 담당 순)
"간단해서 그냥" 생략
신규 에이전트 임의 추가 (사용자 승인 필수)
복합 작업 — 수직 지휘 체계
지시 방향 (위→아래)
사용자 → orchestrator
orchestrator → lead-*
lead-* → 담당 에이전트
검토 방향 (아래→위)
담당 에이전트 → lead-*
lead-* → orchestrator
orchestrator → 사용자
⚠️ STEP 0~3 중 하나라도 빠지면 그 작업은 미완료로 간주. 복합 작업 기준: 에이전트 3개 이상 또는 파이프라인 2단계 이상.
v6.0 추가 규칙: 모달은 ESC 키 및 배경 클릭으로 닫기 지원. Ctrl+K는 에이전트 탭 자동 전환 후 검색창 포커스. Ctrl+/로 단축키 도움말 토스트 즉시 표시. 한자·일본어 사용은 모든 맥락에서 절대 금지. API 에이전트 상태 업데이트는 POST /api/status/{id} 로 직접 호출 가능.
15 · 트러블슈팅 / FAQ
❓ 에이전트 수가 29명으로 표시됩니다
원인: agent_status.json이 구버전이거나 누락된 에이전트가 있음
해결: python scripts/update_status.py orchestrator idle "" 실행 — common_io.py의 _patch_missing_agents()가 33명으로 자동 보완
❓ API 서버 연결 도트가 빨간색입니다
원인: FastAPI 서버(포트 8765)가 실행되지 않음
해결: scripts\start_api.bat 실행. 미실행 시에도 이슈 탭은 localStorage 폴백으로 기본 기능 동작.
❓ 메타버스 스크린샷 버튼이 작동하지 않습니다
원인: Phaser 게임 씬 초기화 전에 버튼 클릭
해결: 페이지 로드 후 캐릭터가 화면에 표시될 때까지 약 2~3초 대기 후 클릭. window.gameScene이 등록된 이후에만 동작.
❓ comms 탭 피드가 로딩 중으로 멈춥니다
원인: API 서버 미실행 or work_log.jsonl 없음
해결: API 연결 시 새로고침 버튼 클릭. API 없으면 localStorage 타임라인 데이터로 폴백 표시됨.
❓ git pull 시 agent_status.json 충돌이 발생합니다
원인: 멀티 PC 환경에서 동시 수정
해결: .gitattributes에 agent_status.json merge=ours 설정 완료 — 로컬 버전이 항상 우선. auto_sync.py도 ours 전략 적용.
❓ validate.py 실행 후 파일이 변경되었습니다
원인: --fix 옵션으로 자동 수정됨
해결: 수정 전 .bak 백업 파일이 자동 생성됨. diff로 변경 내용 확인 후 이상 없으면 commit.
